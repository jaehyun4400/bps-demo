## 🎯 기획

### 배경

QA 포트폴리오에 등재할 가상의 모바일 게임 **"Blades of Plumflower Beneath Snow (BPS)"** 를 직접 기획·설계·구현. 실제 개발팀이 납품한 빌드를 테스트하는 QA 엔지니어의 관점으로 전체 문서를 구성하였으며, 요구사항 작성부터 테스트 케이스 설계, 버그 리포트, 비기능 테스트까지 실무 프로세스를 그대로 재현.

### v1.2.0 업데이트 스펙 기획

**콘텐츠 추가**

| 구분   | 이름              | 등급 | 특이사항                                     |
| ------ | ----------------- | ---- | -------------------------------------------- |
| 캐릭터 | 혜 귀인           | SSR  | 패시브: HP 60% 이하 시 보호막 발동           |
| 무기   | 화씨벽            | SSR  | 마법구슬(orb) 타입, 범위 공격 +25%           |
| 의복   | 대례복            | —    | 무기 고유효과 증폭 (공격력/속도/치명타 증가) |
| 이벤트 | 두근두근 발렌타인 | —    | 몬스터 20마리 처치당 초콜릿 상자 1개 드랍    |

**시스템 변경**
- 이벤트 페이지 추가 (팝업, 재화 교환)
- 상점 UI 개선 (결제 버튼 위치 변경, 팝업 흐름 수정)

### 테스트 전략 설계

**우선순위 분류**

| 우선순위 | 영역               | 주요 검증 항목                                             |
| -------- | ------------------ | ---------------------------------------------------------- |
| P1       | 자동 전투 / 능력치 | 스킬 발동 순서, 무기·의복 고유효과, 전투력 UI vs 실제 결과 |
| P2       | 이벤트 UI / 보상   | 팝업 노출 조건, 페이지 플로우, 보상 데이터 저장, 동기화    |
| P3       | 상점 UI            | 결제 동선, 팝업 겹침, 뒤로가기 상태 유지                   |

**테스트 설계 원칙**
1. **시스템 영향 중심** — 기능 단위보다 기존 시스템 영향도 우선
2. **리스크 기반 우선순위** — 유저 체감 리스크 기준 P1/P2/P3 분류
3. **조건 기반 설계** — 경계값 테스트 (HP 61%=미발동, 60%=발동, 59%=발동)
4. **범위 최소화** — 대표 시나리오 + 영향도 높은 조합 위주
5. **라이브 QA 관점** — 배포 후 모니터링 항목까지 함께 정의

## 🏗️ 설계

### 화면 구조

```
타이틀
  └── 로비
        ├── 던전 선택 → 전장 (필드 액션 전투 / 보스 고정 전투)
        ├── 편성 (캐릭터 / 무기 / 의복)
        ├── 성장 (레벨업 / 능력치 강화)
        ├── 소환 (가챠 / 10연차)
        ├── 이벤트 (두근두근 발렌타인)
        ├── 상점 (패키지 / 다이아 탭)
        ├── 우편함 (메일 보상)
        ├── 업적 (50개 / 6카테고리)
        └── 리딤코드 (7종)
```

### 전투 시스템 설계

| 모드            | 설명                                                |
| --------------- | --------------------------------------------------- |
| 일반 스테이지   | 실시간 필드 액션 — 조이스틱 이동, 4방향 몬스터 스폰 |
| 보스 스테이지   | 좌(플레이어 SD) / 우(보스) 고정 배치                |
| 자동 전투       | ▶/⏸ 버튼으로 토글                                   |
| 배속            | 1x / 2x                                             |
| 반복사냥        | 같은 스테이지 무한 반복 (보스 스테이지 포함)        |
| 자동돌파        | 보스 포함 자동 다음 스테이지 진행                   |
| 백그라운드 전투 | 전투 중 다른 메뉴 이동 시에도 전투 유지             |

### 주요 전투 상수

| 상수                   | 값  | 설명                              |
| ---------------------- | --- | --------------------------------- |
| `PLAYER_SPEED`         | 4.5 | 조이스틱 이동 속도 (px/frame)     |
| `MON_SPEED`            | 2.5 | 몬스터 접근 속도 (px/frame)       |
| `JOY_RADIUS`           | 36  | 조이스틱 최대 반경 (px)           |
| `MELEE_RANGE`          | 80  | 근거리 공격 사거리 (px)           |
| `RANGED_RANGE`         | 170 | 원거리 공격 사거리 (px)           |
| `RANGED_STOP`          | 130 | 원거리 몬스터 접근 정지 거리 (px) |
| `FIELD_SPAWN_MIN_DIST` | 110 | 스폰 최소 거리 (px)               |

### 보상 공식

| 항목                | 공식                        | 비고                     |
| ------------------- | --------------------------- | ------------------------ |
| 클리어 골드 (D1~D4) | `floor(20 × 1.06^n)`        | n = 절대 스테이지 인덱스 |
| 클리어 골드 (D5)    | `floor(20 × 1.5 × 1.06^n)`  | n = 40 + sIdx            |
| 클리어 골드 (D7)    | `floor(20 × 2.0 × 1.06^n)`  | n = 60 + sIdx            |
| 경험치              | `floor(7 × 1.10^n)`         | 동일 인덱스 기준         |
| 처치 골드           | 몬스터 기본 gold × 4        | 전투 중 별도 표시        |
| 강화 비용           | `floor(150 × 1.22^현재단계)` | 캐릭터 능력치 강화 시    |

- **캐릭터 레벨/강화 시스템** — 등급별 최대 레벨(R:40, SR:60, SSR:80), 7종 능력치 강화
- **리딤코드 시스템** — 7종, `S.redeemedCodes` 배열 기반 중복 방지

### Phase 4 · 2026-04-23 — 필드 액션 전투 시스템 대규모 개편

- 정적 좌우 배치 → 실시간 조이스틱 필드 액션 전투로 전환

- 재화 UI 대형 숫자 축약 표시 (100만 / 1000만 / 1억)

### Phase 5 · 2026-04-23 — D6 던전 추가 + 검기 시각 효과

  - bow: `.proj-arrow` CSS transition 화살 이동
  - orb: `.proj-fire` 파이어볼 + 범위 피해

### Phase 6 · 2026-04-24 — D7 던전 + SD 치비 캐릭터 시스템

- **D7 던전 (절로 가는 길)** — 30스테이지 3구역, 5종 신규 드랍 무기 (SR 2종, SSR 3종)
- **업적 시스템 대폭 확장** — 4카테고리 → 6카테고리(수집/도전 추가), 17개 → 50개
- **SD 치비 캐릭터 시스템 (필드 전투)**
  - 무기 종류에 따라 이미지 전환 (sword_girl / bow_girl / mage_girl / normal_girl)
  - 등급별 오라 효과 (SSR: 금빛 / SR: 보라 / R: 파랑 drop-shadow 펄스)
  - 적 위치 기준 좌우 반전 (`updateSDFlip()`, 매 40ms 갱신)
- **SD 치비 캐릭터 시스템 (보스 전투)**
  - 보스전 SD 적용, 필드 대비 10% 크기 증가 (45px → 50px)
  - 캐릭터·무기 변경 시 보스전 중에도 즉시 반영
- **근거리 무기 vs 원거리 몬스터 버그 수정**
  - 원인: 원거리 몬스터 정지 거리(130px) > 근거리 공격 범위(80px)
  - 수정: `S.playerTargetIdx` 락온 타겟 추가, `pickPlayerTarget()` 함수 구현
  - 조이스틱 미조작 + 근거리 무기 + 타겟이 사정거리 밖인 경우 자동 추적 이동

### Phase 7 · 2026-04-29 — HP 오브 UI 개편 + 혜 귀인 보호막 시각화 + 배포

- **HP 오브 UI 전면 교체 — 디아블로 스타일 액체 구체**
  - 기존 `conic-gradient` 도넛 차트(56px) → Canvas 기반 액체 구체(72px)
  - `requestAnimationFrame` 루프 + `Math.sin()` 파도 애니메이션
  - HP 비율 연동 색상: 빨강(50%↑) / 주황(25%↑) / 진빨강(위험)
  - 유리 광택 + 내부 림 어두운 그라디언트로 입체감 구현
- **혜 귀인 보호막 시각화 완성**
  - 오브 내부: 쉴드 수치만큼 HP 액체 위에 하늘색 액체 레이어로 시각화
  - 외곽 링: 활성=파란 펄스 / 쿨타임=회색 dim / 소진=shatter 파열 애니메이션
  - 보스 모드 AP HP 바에도 파란색 쉴드 구간 추가 (`ap-shield-fill`)
- **혜 귀인 보호막 로직 완성**
  - HP 60% 이하 시 최대 HP 15% 보호막 발동 (쉴드 선차감 방식)
  - 소진 후 쿨타임 20초 / 스테이지 이동 시 초기화
  - 쿨타임 종료 시 HP 60% 이하이면 즉시 자동 재발동
  - 스테이지 진입 즉시 HP 60% 이하 확인 → 즉시 발동
- **자동돌파 버그 수정**
  - D1~D6 보스 클리어 후 자동돌파가 멈추는 문제 수정
  - `isLast` 조건을 "마지막 던전의 마지막 스테이지"일 때만 true로 교정
- **GitHub Pages 배포 + CLAUDE.md 생성**
  - 라이브 데모: https://jaehyun4400.github.io/bps-demo/
  - `CLAUDE.md` 생성 — 세션 간 메모리 유실 방지용 작업 가이드
