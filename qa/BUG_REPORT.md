## BUG-001

### 기본 정보

| 항목 | 내용 |
|---|---|
| Bug ID | BUG-001 |
| 제목 | 이벤트 재화 교환 연속 터치 시 보상 중복 지급 |
| 심각도 | Critical |
| 우선순위 | P1 |
| 발견일 | 2026-04-22 |
| 발견자 | 박재현 |
| 상태 | Open |
| 연결 TC | TC-007 (FAIL) |

### 환경

| 항목 | 내용 |
|---|---|
| 기기 | 갤럭시 S24 (Android 14) |
| 빌드 | v1.2.0-rc1 |
| 서버 | Staging |

### 재현 절차

1. 로비 → 이벤트 탭 진입
2. '두근두근 발렌타인' 이벤트 페이지 진입
3. 초콜릿 상자 보유 상태에서 재화 교환 버튼을 빠르게 연속으로 탭 (0.2초 이내 3회 이상)
4. 지급된 재화 수량 확인

### 기대 결과

보상이 1회만 지급되며, 연속 터치에 대한 중복 처리 방지

### 실제 결과

터치 횟수만큼 보상이 중복 지급됨 (3회 탭 시 3배 보상)

### 원인 분석

서버에 교환 요청을 전송한 후 응답을 받기 전에 버튼이 재활성화되어 중복 요청이 발생함. 클라이언트 측 버튼 Disable 처리가 누락된 상태로, 서버의 중복 방지 로직도 부재한 것으로 추정됨.

### 영향 범위

이벤트 재화 교환 화면 전체. 단기간에 과도한 재화 지급으로 게임 경제 밸런스 붕괴 가능.

### 조치 방향

- 교환 버튼 클릭 시 즉시 비활성화(Disable) 처리
- 서버 응답 완료 후 버튼 재활성화
- 서버 측 동일 요청 중복 처리 방지 로직 추가 검토

- 스크린샷: `attachments/BUG-001_before.png`, `attachments/BUG-001_after.png`

## BUG-002

### 기본 정보

| 항목 | 내용 |
|---|---|
| Bug ID | BUG-002 |
| 제목 | 대례복 장착 후 상태창 전투력 UI 미갱신 |
| 심각도 | Major |
| 우선순위 | P1 |
| 발견일 | 2026-04-22 |
| 발견자 | 박재현 |
| 상태 | Open |
| 연결 TC | TC-004 (FAIL) |

### 환경

| 항목 | 내용 |
|---|---|
| 기기 | 갤럭시 S24 (Android 14), 아이폰 14 Pro (iOS 17) |
| 빌드 | v1.2.0-rc1 |
| 서버 | Staging |

### 재현 절차

1. 편성 탭 → 의복 슬롯 진입
2. 대례복을 장착
3. 장착 직후 상태창(전투력 수치) 확인
4. 상태창을 나갔다가 재진입 후 수치 재확인

### 기대 결과

대례복 장착 즉시 전투력 수치가 갱신됨

### 실제 결과

- 장착 직후: 전투력 수치가 이전 값으로 그대로 표시됨
- 상태창 재진입 후: 갱신된 전투력 수치 정상 표시

### 원인 분석

실제 능력치 계산은 장착 즉시 정상 적용되고 있으나, 상태창 UI 렌더 함수가 의복 장착 이벤트에 연결되어 있지 않아 갱신이 누락된 것으로 추정됨 (화면 재진입 시 정상 표시되는 점에서 렌더 트리거 누락으로 판단).

### 영향 범위

의복 장착 후 상태창 전투력 수치 표시에만 영향. 실제 전투 데미지 계산에는 영향 없음. 단, 유저가 표시된 전투력을 기준으로 의사결정을 하는 만큼 신뢰도에 영향을 줌.

### 조치 방향

- 의복 장착 완료 이벤트에 상태창 UI 렌더 함수(`renderStats()`) 연결
- 장착/해제 모든 경우 갱신 여부 추가 확인

### 재현 가능 여부

데모에서 재현 가능 — 편성 탭에서 대례복 장착 후 상태창 전투력 수치 즉시 확인

- 스크린샷(재진입 후): `attachments/BUG-002_after.png`
