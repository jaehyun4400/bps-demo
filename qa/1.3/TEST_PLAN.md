## 1. 개요

### 1.1 목적

BPS v1.3.0 업데이트에 포함된 신규 라이브 콘텐츠(출석 이벤트, 픽업 배너)에 대해 출시 전 테스트를 수행하여 결함을 조기에 발견하고, 재화/확률 정합성을 보장한다.

### 1.2 배경

v1.3.0은 신규 캐릭터 없이 **운영성 콘텐츠** 중심의 업데이트이다. '여름방학맞이 7days 특별 출석이벤트'와 기존 SSR '혜 귀인'을 활용한 **픽업 소환 배너**를 도입한다. 두 콘텐츠 모두 **재화 지급 및 확률 시스템**에 직접 연관되므로, 보상 정합성·중복 방지·천장 동작을 집중 검증한다.

## 2. 테스트 범위

### 2.1 In Scope (테스트 대상)

| 구분 | 항목 |
|---|---|
| 신규 콘텐츠 | 출석 이벤트 (7일 누적, 1일 1회 수령, 자동 팝업, 원형 아이콘) |
| 신규 콘텐츠 | 출석 보상 지급 정합성 (Day 1~7 골드/다이아) |
| 신규 콘텐츠 | 혜 귀인 픽업 배너 (확률 UP, 천장 50회, 1회/10+1 소환) |
| 회귀 | 기존 소환 시스템 (`pullOne`, 천장 카운터, 조각 지급) |
| 회귀 | 재화(골드/다이아) 합산 및 저장(localStorage) |

### 2.2 Out of Scope (테스트 제외)

| 항목 | 제외 이유 |
|---|---|
| 서버 인프라 / 결제 실거래 | 별도 팀·PG사 연동 테스트 |
| 로컬라이징(다국어) | v1.3.0 대상 언어 변경 없음 |
| v1.2.0 이전 기능 | 변경 없음, 기존 검증 완료 (회귀 최소 범위만) |

## 3. 테스트 전략

### 3.1 테스트 유형

| 유형 | 설명 | 비중 |
|---|---|---|
| 기능 테스트 | 출석/소환 정상 동작, 보상 지급 | 55% |
| 경계값 테스트 | 사이클 경계(Day 7→1), 천장(49→50), 날짜 변경 | 30% |
| 회귀 테스트 | 기존 소환/재화/저장 영향도 | 15% |

### 3.2 우선순위 분류

| 우선순위 | 영역 | 판단 기준 |
|---|---|---|
| P1 | 사이클 리셋 / 소환 수량 / 천장 / 확률 | 재화 손실·확률 정합성 직결 |
| P2 | 보상 정확성 / 중복 방지 / 자동 팝업 | 데이터 정합성, 유저 체감 |

### 3.3 설계 원칙

1. **재화·확률 정합성 우선** — 지급 수량/확률은 데이터 기반 검증
2. **경계값 집중** — 사이클 전환(Day 7→1), 천장(49→50), 자정 경과
3. **폴더 단위 자산 관리** — v1.3.0 TC는 본 폴더의 TEST_CASES.md(TC-001~013)에 독립 관리
4. **라이브 운영 관점** — 장기 접속 유저 시나리오(7일 완주) 포함
