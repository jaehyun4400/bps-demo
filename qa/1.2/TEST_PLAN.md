# 테스트 계획서 (Test Plan)

**프로젝트**: Blades of Plumflower Beneath Snow (BPS)  
**버전**: v1.2.0  
**작성자**: 박재현  
**작성일**: 2026-04-17  
**최종 수정일**: 2026-04-29  

---

## 1. 개요

### 1.1 목적

BPS v1.2.0 업데이트에 포함된 신규 콘텐츠 및 시스템 변경 사항에 대해 출시 전 테스트를 수행하여 결함을 조기에 발견하고, 서비스 품질을 보장한다.

### 1.2 배경

v1.2.0은 신규 SSR 캐릭터 '혜 귀인' 추가 및 이와 연동된 보호막 패시브 시스템, 이벤트 페이지 신규 도입, 상점 UI 개선을 포함하는 대규모 업데이트이다. 기존 전투 시스템 및 상점 결제 흐름에 영향을 주는 변경이 포함되어 있어 회귀 테스트를 병행한다.

---

## 2. 테스트 범위

### 2.1 In Scope (테스트 대상)

| 구분 | 항목 |
|---|---|
| 신규 콘텐츠 | 혜 귀인 패시브 (보호막 시스템) |
| 신규 콘텐츠 | 신규 무기 화씨벽 (전체 공격 스플래시) |
| 신규 콘텐츠 | 신규 의복 대례복 (무기 고유효과 증폭) |
| 신규 콘텐츠 | 이벤트 '두근두근 발렌타인' (팝업, 드랍, 재화 교환) |
| 시스템 변경 | 상점 UI 개선 (결제 버튼 위치, 팝업 흐름) |
| 회귀 | 기존 전투 시스템 (자동전투, 배속, 반복사냥, 자동돌파) |
| 회귀 | 캐릭터 성장 (레벨업, 능력치 강화) |
| 회귀 | 소환 시스템 |
| 회귀 | 업적 / 우편 / 리딤코드 |

### 2.2 Out of Scope (테스트 제외)

| 항목 | 제외 이유 |
|---|---|
| 서버 인프라 성능 테스트 | 별도 인프라 팀 담당 |
| 결제 실 거래 검증 | PG사 연동 테스트로 별도 진행 |
| 로컬라이징(다국어) | v1.2.0 대상 언어 변경 없음 |
| v1.1.x 이전 레거시 기능 | 변경 없음, 기존 검증 완료 |

## 3. 테스트 전략

### 3.1 테스트 유형

| 유형 | 설명 | 비중 |
|---|---|---|
| 기능 테스트 | 각 기능의 정상 동작 여부 확인 | 60% |
| 경계값 테스트 | HP %, 수량, 쿨타임 등 경계 조건 집중 | 20% |
| 회귀 테스트 | 기존 기능 영향도 확인 | 15% |
| 비기능 테스트 | 기기별 성능, 발열, FPS, 메모리 | 5% |

### 3.2 우선순위 분류

| 우선순위 | 영역 | 판단 기준 |
|---|---|---|
| P1 | 전투 / 능력치 / 보호막 | 유저 체감 직결, 데이터 정합성 |
| P2 | 이벤트 UI / 보상 | 한정 기간 콘텐츠, 데이터 저장 |
| P3 | 상점 UI | 결제 동선, 팝업 흐름 |

### 3.3 설계 원칙

1. **시스템 영향 중심** — 기능 단위보다 기존 시스템 영향도 우선 검증
2. **리스크 기반 우선순위** — 유저 체감 및 수익 영향도 기준 P1/P2/P3 분류
3. **조건 기반 설계** — 경계값 분석 적극 활용 (HP 61%=미발동, 60%=발동)
4. **범위 최소화** — 대표 시나리오 + 영향도 높은 조합 위주
5. **라이브 QA 관점** — 배포 후 모니터링 항목까지 함께 정의

## 7. 리스크 및 대응 방안

| 리스크 | 영향도 | 대응 방안 |
|---|---|---|
| 보호막 시스템이 기존 전투 로직과 충돌 | High | P1 TC 우선 실행, 전투 전체 회귀 |
| 이벤트 재화 교환 중복 지급 | High | 서버 요청 중복 방지 로직 추가 요청 |
| 대례복 UI 미갱신 시 유저 혼란 | Medium | 수정 확인 후 출시, 미수정 시 릴리즈 노트 안내 |
| 신규 기기에서 Canvas HP 오브 렌더 이슈 | Medium | 비기능 테스트에서 기기별 확인 |
| 일정 지연으로 회귀 테스트 축소 | Low | P1 위주 최소 회귀 범위 사전 정의 |

## 8. 완료 기준 (Exit Criteria)

- P1 TC 전체 PASS
- P2 TC PASS율 90% 이상
- Critical / High 결함 0건 미해결
- 비기능 테스트 기준치 달성 (FPS 30+, 메모리 증가율 20% 미만)
- 테스트 완료 보고서 제출 및 PM 승인
