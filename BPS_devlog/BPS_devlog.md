## 2026-04-29 — GitHub 배포 + 자동돌파 버그 수정 + 보호막 기획

### GitHub 배포

개발 파일을 친구에게 공유했더니 이미지가 엑스박스(깨진 이미지)로 보이는 문제가 있었다. HTML 파일만 단독으로 전달하면 `portraits_unified/`, `sd_chars/` 같은 상대경로 폴더가 없으니 당연한 일이었다.

해결책으로 GitHub Pages를 이용한 무료 정적 호스팅을 선택했다. 회사 GitHub 계정에 개인 레포를 만드는 것에 대한 우려가 있었는데, GitHub 조직(org) 구조상 개인 레포와 회사 org 레포는 완전히 분리되어 있어 문제없음을 확인했다. 또한 회사 컴퓨터이기 때문에 git config를 `--global`이 아닌 `--local`로 설정해 bps 폴더에만 적용되도록 했다.

배포 시 `.gitignore`에서 루트 경로 지정 없이 파일명만 기재했더니, `portraits_unified/` 폴더 안의 동명 파일까지 제외되는 문제가 발생했다. `/gungny.png` 형식으로 루트 경로를 명시해 해결했다.

### 자동돌파(autoDive) 버그 수정

D2-10(던전2 보스), D3-10(던전3 보스) 클리어 후 자동돌파가 멈추는 버그가 발생했다.

**원인 분석:**

`showResult()` 함수 내 `isLast` 계산 로직이 문제였다.

```javascript
// 버그 코드
const isLast = S.stage === (DUNGEONS[S.dungeon].stageCount || 10) - 1;
```

D1~D4는 `stageCount`가 없어 기본값 10이 적용된다. 따라서 stage === 9 (보스 스테이지)이면 `isLast = true`가 되어버렸고, 자동돌파 블록인 `if(win && S.autoDive && !isLast)` 조건이 항상 false가 되어 실행되지 않았다.

**수정:**

```javascript
// 수정 코드
const isLastStage = S.stage === (DUNGEONS[S.dungeon].stageCount || 10) - 1;
const isLast = isLastStage && S.dungeon === DUNGEONS.length - 1;
```

마지막 스테이지이면서 **동시에 마지막 던전**일 때만 `isLast = true`가 되도록 수정했다. D7 마지막 스테이지에서만 로비로 복귀하게 되었다.

### 보호막 시스템 기획 논의

혜 귀인의 패시브 "HP 60% 이하 시 보호막 발동"에 대한 상세 설계를 논의했다. 주요 결정 사항:

- **보호막 수치:** 캐릭터 최대 체력의 15% 고정
- **데미지 흡수 방식:** 쉴드가 HP보다 먼저 차감 (버퍼 역할)
  - 예) HP 80%, 쉴드 15% 활성 → 70% 데미지 수신 → 쉴드 15% 선차감 → HP에 55% 적용 → 최종 HP 25%
- **재발동 쿨타임:** 20초
- **쿨타임 경과 시:** HP가 60% 이하이면 즉시 자동 재발동
- **쉴드 캡:** 쿨타임 경과 시 잔여 쉴드 수치와 무관하게 15%로 리셋 (누적 불가)
- **스테이지 진입 시:** 쿨타임 초기화, HP 60% 이하면 즉시 발동
- **HP 강화와 연동:** 의도된 설계 — HP를 올릴수록 쉴드량도 함께 증가, 유저에게 "강공 vs 버티기" 선택권 부여

## 2026-04-24 — D7 던전 + SD 치비 캐릭터 시스템 + 근거리 버그 수정

### D7 던전 — 절로 가는 길 🛕

D6까지 구현된 상태에서 D7을 추가했다. D5와 동일한 30스테이지 3구역 섹션 구조를 사용했으며, D7에 맞게 보상 공식을 별도 분기로 추가했다(`mult=2.0, n=60~89`).

D7 드랍 무기 5종을 새로 설계했다. 특히 **뇌음목**은 기존에 없던 "확률 폭발" 효과(`explodeChance: 0.15`)를 가진 첫 번째 무기였다. 15% 확률로 추가 범위 피해를 주는 메커니즘으로, 전투 루프에 `explodeChance` 분기를 별도로 추가해 처리했다.

또한 D5~D7까지의 섹션 구조를 지원하기 위해 기존에 D5 전용으로 하드코딩되어 있던 `selectDungeon()`, `countDunClears()`, `isDungeonCleared()`, `autoNextStage()`, `isLast` 조건을 `DUNGEONS[dIdx].stageCount || 10` 기반으로 일반화했다.

### 업적 시스템 대폭 확장

기존 4개 카테고리(전투/던전/소환/성장) 17개에서 **수집**, **도전** 카테고리를 추가해 총 **6개 카테고리 50개**로 확장했다.

- 수집: 무기/의복 보유 수, 리딤코드 사용
- 도전: 총 스테이지 클리어 수 누적

### SD 치비 캐릭터 시스템

필드 전투 화면에서 기존 초상화(portrait) 대신 SD 치비 캐릭터 이미지를 사용하도록 개선했다. 무기 종류에 따라 이미지가 자동으로 전환되고, 적의 위치에 따라 좌우 반전도 자동으로 처리된다.

이미지 배경 제거 작업이 예상보다 까다로웠다. Python Pillow의 flood-fill 방식을 사용했는데, tolerance를 높이면 캐릭터 팔/손 영역까지 제거되고, 낮추면 배경이 남는 문제를 반복해서 조율해야 했다. 최종적으로 `tolerance=55` + HLS 엣지 정리(L>0.82, S<0.35, 투명 이웃 필요) + 소형 컴포넌트 제거(size<500 or avg_L>0.76) 조합으로 해결했다.

구현 중 두 가지 추가 버그가 발생했다:

- **좌우 반전 시 흰색 깜빡임:** CSS `transition:transform`과 피격 시 `brightness` 효과가 충돌해서 발생. transition 제거 및 brightness 오버라이드 삭제로 해결.
- **캐릭터 변경 시 오라 미반영:** `updateSDChar()`가 이미지 경로만 갱신하고 오라 클래스(SSR/SR/R에 따른 glow 효과)는 갱신하지 않던 문제. 오라 클래스와 이름 텍스트를 함께 갱신하도록 함수를 확장해 해결.

보스 전투에도 SD 캐릭터를 적용했다. 필드(45px)보다 10% 크게(50px) 표시하며, 캐릭터/무기 변경 핸들러에서 `isFieldMode()` 조건을 제거해 보스전 중에도 즉시 반영되도록 했다.

### 근거리 무기 vs 원거리 몬스터 버그 수정

근거리 무기 착용 시 원거리 몬스터를 공격할 수 없는 버그가 있었다.

**원인:** 원거리 몬스터는 플레이어로부터 130px(RANGED_STOP) 거리에서 멈추는데, 근거리 공격 범위는 80px(MELEE_RANGE)이었다. 플레이어가 80px 안으로 들어가지 않으면 공격 자체가 불가능한 구조였다.

**해결:** `S.playerTargetIdx` 상태 필드를 추가해 락온 타겟을 관리하도록 했다. `pickPlayerTarget()` 함수를 구현해 현재 타겟이 살아있으면 유지하고, 사망 시 가장 가까운 적으로 전환한다. `fieldUpdate()` 루프에서 조이스틱 미조작 + 근거리 무기 + 타겟이 MELEE_RANGE 밖인 경우 플레이어가 자동으로 타겟을 향해 이동하도록 추가했다.

## 2026-04-23 — 필드 액션 전투 대규모 개편 + D6 던전 + 검기 효과

### 정적 전투 → 실시간 필드 액션 전투로 전환

기존에는 좌(플레이어)/우(몬스터) 고정 배치로 전투가 진행됐다. 이를 조이스틱으로 이동하고 몬스터가 실시간으로 접근해오는 필드 액션 방식으로 전면 개편했다. 이 결정이 코드 구조상 가장 큰 변경이었다.

**초기 스폰 편향 버그:** 구현 초기에 몬스터가 화면 상단(10시/2시 방향)에만 집중해서 나타나는 문제가 있었다. 원인은 스폰 로직이 상단 경계에만 집중되어 있었기 때문이다. 아레나 4방향(상/하/좌/우) 경계에서 균등하게 랜덤 스폰되도록 변경해 해결했다.

**몬스터 AI 미작동 버그:** `fieldUpdate()` 함수 상단에 `if(!S.running)return` 가드가 있었는데, 자동전투 버튼을 누르기 전(S.running=false) 상태에서는 몬스터 AI가 아예 동작하지 않았다. `if(!_inBattle)return`으로 변경해 전투 세션이 존재하는 한 항상 AI가 동작하도록 수정했다.

전투 중 다른 메뉴(편성, 상점 등)로 이동해도 전투가 백그라운드에서 계속 진행되는 기능도 함께 추가했다. `_inBattle` 플래그와 `navFromBattle()` 함수로 구현했다.

### D6 던전 — 고성 🏯

10스테이지 구조로 추가했다. 보스는 "성주"이며, 드랍 무기 "귀환도(SR)"를 5% 확률로 드랍한다.

### 검기(劍氣) 시각 효과

sword 타입 무기 공격 시 타겟 위치에 흰색/하늘색 광선 슬래시 애니메이션이 재생되도록 추가했다. `.slash-fx`(주 광선)와 `.slash-fx2`(보조 광선) 두 레이어로 구성, 0.32초 동안 재생 후 제거된다.

이를 계기로 무기 타입별 공격 시각 효과를 체계화했다:
- sword: 검기 슬래시
- bow: 화살 날아감 (`.proj-arrow`)
- orb: 파이어볼 날아감 + 범위 피해 (`.proj-fire`)

### 리딤코드 시스템

7종의 리딤코드를 구현했다. `S.redeemedCodes` 배열에 사용 이력을 저장해 계정당 1회만 사용 가능하도록 중복 방지 처리를 했다.

재화 UI에 대형 숫자 축약 표시(100만 / 1000만 / 1억)도 함께 추가했다. 처치 골드는 기본값 대비 × 4배로 상향해 게임 속도감을 높였다.

## 2026-04-21 — 고급 기능 확장

### D5 던전 — 깊은 숲 🌲

기존 D1~D4의 단순 10스테이지 구조에서 벗어나, 30스테이지 3구역 섹션 구조를 처음으로 도입했다. 구역마다 별도의 보스와 SSR 드랍 무기를 배치해 진행의 동기 부여를 강화했다.

이 과정에서 기존에 `stageCount: 10`으로 하드코딩되어 있던 던전 스테이지 수 처리를 `DUNGEONS[dIdx].stageCount || 10` 동적 처리로 일반화했다.

### 업적 시스템 초기 구현

전투/던전/소환/성장 4개 카테고리로 처음 구현했다. 수동 수령 방식을 채택했다 (달성해도 자동 지급 없이 유저가 직접 수령).

### 캐릭터 레벨/강화 시스템

등급별 최대 레벨(R: 40, SR: 60, SSR: 80)과 7종 능력치 강화(힘/체력/방어력/치명타확률/치명타피해/회피/공격속도)를 구현했다.

경험치 포션 사용 또는 던전 클리어 시 자동으로 경험치가 적용된다. 강화 비용은 `floor(200 × 1.3^현재단계)` 공식을 사용해 지수적으로 증가한다.

### 스테이지 클리어 기록 시스템

`S.stageClears` 객체에 `"dIdx-sIdx": true` 형태로 클리어 기록을 저장한다. 스테이지 그리드 UI에서 클리어한 셀에 녹색 테두리와 ✓ 표시가 나타난다.

## 2026-04-17 — 게임 기획 및 세계관 설계

### 프로젝트 시작 배경

QA 엔지니어 포트폴리오를 위해 가상의 모바일 게임을 직접 기획하고 데모로 구현하는 프로젝트를 시작했다. 단순히 TC 문서만 작성하는 것보다, 실제로 동작하는 데모와 함께 제공하면 훨씬 설득력 있는 포트폴리오가 될 것이라 판단했다.

### 게임 기본 설계

장르는 **동양 궁중 판타지 모바일 수집형 방치 RPG**로 결정했다. 전투는 Fully Automated Battle System을 채택해 유저가 직접 조작하지 않아도 전투가 진행되는 방치형 구조로 갔다.

가상 플랫폼은 Android/iOS이며, 주요 수익 모델은 캐릭터/무기 가챠와 인앱 결제 패키지로 설정했다.

### v1.2.0 업데이트 스펙 정의

포트폴리오 문서의 기준이 되는 업데이트 버전을 v1.2.0으로 설정했다. 이번 업데이트의 핵심 콘텐츠는 다음과 같다:

- **혜 귀인 (SSR):** HP 60% 이하 시 보호막 발동 패시브
- **화씨벽 (SSR):** 마법구슬 타입, 범위 공격 +25%
- **대례복:** 무기 고유효과 증폭
- **두근두근 발렌타인 이벤트:** 초콜릿 재화 수집 및 교환

테스트 전략의 핵심은 기능 단위보다 **시스템 영향도** 중심으로 우선순위를 잡는 것이었다. 혜 귀인 패시브의 경계값(HP 61%=미발동, 60%=발동)처럼 조건 기반 테스트를 중심으로 TC를 설계했다.
