## 2. 진입 경로

- **자동 팝업**: 게임 접속(로비 진입) 시 당일 미수령이면 자동으로 1회 노출
- **수동 진입**: 로비 화면 **우측 상단 원형 아이콘(📅 출석)** 클릭
- **미수령 표시**: 원형 아이콘 우상단 빨간 뱃지

---

## 3. 정상 수령 방식 (실제 플레이)

출석은 **하루 1회, 날짜가 바뀌어야** 다음 Day로 진행됩니다.

- 수령 가능 조건: `오늘 날짜 ≠ 마지막 수령 날짜` (`canCheckInToday()`)
- 오늘 Day 1 수령 → **다음 날 접속**하면 Day 2 버튼 활성화 → 수령
- 위를 7일 반복하면 Day 7까지 완료
- 하루 빠져도 누적 진행 (이어서 다음 Day부터 수령)

---

## 4. 데모/테스트에서 Day 2~7 빠르게 받는 방법

날짜를 바꾸지 않고 한 자리에서 다음 Day를 확인하려면, 브라우저 콘솔(F12)에서:

```js
// 오늘 다시 수령 가능하게 만들기 (마지막 수령 날짜 초기화)
S.lastCheckIn = '';
renderCheckIn();          // 수령 버튼 재활성화
```

이후 팝업에서 **"오늘 보상 수령"** 클릭 → 다음 Day 보상이 지급됩니다.
이 과정을 반복하면 Day 2 → 3 → … → 7까지 연속 확인 가능합니다.

### 특정 Day로 바로 점프

```js
// 예: Day 6부터 받기 (checkInDay = 완료 일수 = 다음 수령 슬롯 인덱스)
S.checkInDay = 5;         // 0=Day1, 5=Day6
S.lastCheckIn = '';
renderCheckIn();
```

> `S.checkInDay` 값 = 현재까지 완료한 일수 = 다음에 받을 슬롯의 0-based 인덱스
> (0 → Day1 수령 대기, 6 → Day7 수령 대기)

## 5. BUG-001 재현 방법 (사이클 리셋 불가)

> 상세 리포트: `qa/1.3/BUG_REPORT.md` 의 BUG-001 (연결 TC-001)

브라우저 콘솔(F12)에서:

```js
// 1) Day 6까지 받은 상태로 점프 후 Day 7 수령
S.checkInDay = 6; S.lastCheckIn = ''; renderCheckIn();
// → 팝업에서 "오늘 보상 수령" 클릭 (Day7 다이아 25,000 지급, checkInDay=7)

// 2) "다음 날"이 된 상황 흉내 (마지막 수령일을 과거로)
S.lastCheckIn = '2000-01-01'; renderCheckIn();
```

### 기대 vs 실제

| 구분 | 내용 |
|---|---|
| 기대 | 새 사이클로 리셋되어 Day 1 버튼 활성화 ("0/7일") |
| 실제 | "7/7일 완료" 고착, 수령 가능일인데 버튼 **"수령 완료" 비활성화** → 출석 보상 영구 차단 |

### 원인 (한 줄 요약)

`claimCheckIn()`에서 Day 7 수령 후 `checkInDay = 0` 리셋 코드가 누락되어,
버튼 활성 조건 `checkInDay < 7`이 영구히 거짓이 됨.

## 6. 관련 함수 / 상태

| 항목 | 설명 |
|---|---|
| `S.checkInDay` | 완료 일수 = 다음 수령 슬롯 인덱스 (0~7), 영속 저장 |
| `S.lastCheckIn` | 마지막 수령 날짜 'YYYY-MM-DD', 영속 저장 |
| `canCheckInToday()` | 오늘 수령 가능 여부 |
| `openCheckIn()` / `renderCheckIn()` | 팝업 열기 / 그리드·버튼 렌더 |
| `claimCheckIn()` | 오늘 보상 수령 (⚠ BUG-001[v1.3.0] 리셋 누락 지점) |
| `updateCheckInBadge()` | 원형 아이콘 미수령 뱃지 표시/숨김 |
